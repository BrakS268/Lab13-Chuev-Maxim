# Задание 1: Определение агентов и их ролей

## Предметная область: Система управления гостиницей

---

## Агент 1: CheckInAgent (Агент заселения/выселения)

**Роль:** Управляет процессом заселения и выселения гостей, назначает номера.

**Подписка на топик NATS:** `hotel.checkin`

**Входные данные (JSON):**
```json
{
  "task_id": "uuid",
  "type": "check_in" | "check_out",
  "guest_name": "Иван Иванов",
  "room_number": 101,
  "nights": 3,
  "check_in_date": "2025-05-21"
}
```

**Выходные данные (JSON):**
```json
{
  "task_id": "uuid",
  "success": true,
  "output": "Гость Иван Иванов заселён в номер 101 на 3 ночи",
  "room_status": "occupied" | "free"
}
```

**Бизнес-правила:**
- Нельзя заселить гостя в занятый номер
- При выселении номер переводится в статус "требует уборки"
- Минимальный срок проживания — 1 ночь
- Номера: 101–110 (стандарт), 201–205 (люкс)

---

## Агент 2: CleaningAgent (Агент уборки)

**Роль:** Управляет уборкой номеров, назначает задачи персоналу.

**Подписка на топик NATS:** `hotel.cleaning`

**Входные данные (JSON):**
```json
{
  "task_id": "uuid",
  "type": "clean_room" | "inspect_room",
  "room_number": 101,
  "priority": "normal" | "urgent"
}
```

**Выходные данные (JSON):**
```json
{
  "task_id": "uuid",
  "success": true,
  "output": "Номер 101 убран и готов к заселению",
  "room_status": "clean" | "dirty" | "inspecting"
}
```

**Бизнес-правила:**
- После каждого выселения номер обязательно убирается
- Уборка urgent выполняется в первую очередь
- После уборки обязательна инспекция
- Уборка занимает от 30 минут (стандарт) до 60 минут (люкс)

---

## Агент 3: GuestRequestAgent (Агент обработки запросов гостей)

**Роль:** Обрабатывает запросы гостей: еда, полотенца, такси, будильник и т.д.

**Подписка на топик NATS:** `hotel.requests`

**Входные данные (JSON):**
```json
{
  "task_id": "uuid",
  "type": "room_service" | "towels" | "taxi" | "wake_up_call",
  "room_number": 101,
  "guest_name": "Иван Иванов",
  "details": "Завтрак в 8:00, кофе и омлет"
}
```

**Выходные данные (JSON):**
```json
{
  "task_id": "uuid",
  "success": true,
  "output": "Запрос принят: завтрак будет доставлен в номер 101 в 8:00",
  "estimated_time_minutes": 30
}
```

**Бизнес-правила:**
- Обслуживание номеров только с 7:00 до 23:00
- Такси заказывается минимум за 15 минут до отправления
- Запросы на полотенца передаются CleaningAgent
- Гость должен быть заселён (нельзя принять запрос от незаселённого гостя)

---

## Агент 4: BillingAgent (Агент биллинга)

**Роль:** Ведёт учёт расходов гостей, формирует счета, обрабатывает оплату.

**Подписка на топик NATS:** `hotel.billing`

**Входные данные (JSON):**
```json
{
  "task_id": "uuid",
  "type": "add_charge" | "generate_invoice" | "process_payment",
  "guest_name": "Иван Иванов",
  "room_number": 101,
  "amount": 2500.00,
  "description": "Проживание 3 ночи",
  "currency": "RUB"
}
```

**Выходные данные (JSON):**
```json
{
  "task_id": "uuid",
  "success": true,
  "output": "Счёт сформирован: итого 7500.00 RUB",
  "total_amount": 7500.00,
  "invoice_id": "INV-2025-0042"
}
```

**Бизнес-правила:**
- Оплата проживания списывается автоматически при выселении
- Дополнительные услуги (room service) добавляются к счёту сразу
- Счёт формируется только при выселении
- Поддерживаемые валюты: RUB, USD, EUR

## Схема взаимодействия агентов

```
Оркестратор (Python)
    │
    ├──► hotel.checkin   ──► CheckInAgent (Go)
    │                             │
    │                             └──► при выселении → hotel.cleaning
    │
    ├──► hotel.cleaning  ──► CleaningAgent (Go)
    │
    ├──► hotel.requests  ──► GuestRequestAgent (Go)
    │                             │
    │                             └──► запрос полотенец → hotel.cleaning
    │
    └──► hotel.billing   ──► BillingAgent (Go)
                                  │
                                  └──► при выселении ← hotel.checkin
```

## Типы агентов по классификации MAS

| Агент              | Тип          | Описание                                              |
|--------------------|--------------|-------------------------------------------------------|
| CheckInAgent       | Целевой      | Достигает цели: корректное заселение/выселение        |
| CleaningAgent      | Реактивный   | Реагирует на события: номер освободился → убрать      |
| GuestRequestAgent  | Реактивный   | Реагирует на входящие запросы гостей                  |
| BillingAgent       | BDI          | Убеждение: баланс гостя; Желание: закрыть счёт; Намерение: списать оплату |
